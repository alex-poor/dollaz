# Categories & rules

**Categories** are the buckets your spending and income roll up into. Click the colour swatch to recolour, change the type (expense / income / transfer), rename or delete.

**Rules** map a piece of a transaction description to a category. A rule matches when its text appears anywhere in the description (case-insensitive), so `WOOLWORTHS` catches `WOOLWORTHS 1234 SYDNEY`. The most specific (longest) matching rule wins.

- Rules are created automatically as you categorise imports, or add them here.
- **Re-apply rules** re-runs every rule over all transactions — useful after editing rules.
- Deleting a category un-categorises its transactions and removes its rules.
